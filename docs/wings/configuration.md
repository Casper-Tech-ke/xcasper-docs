# ⚙ Wings Configuration

Wings is configured via `/etc/pterodactyl/config.yml`. This file is automatically generated by `wings configure` but you can also edit it manually.

---

## Key Configuration Options

```yaml title="/etc/pterodactyl/config.yml (structure)"
debug: false
uuid: <NODE_UUID>
token_id: <TOKEN_ID>
token: <DAEMON_TOKEN>
api:
  host: 0.0.0.0
  port: 8080
  ssl:
    enabled: true
    cert: /etc/letsencrypt/live/node.yourdomain.com/fullchain.pem
    key:  /etc/letsencrypt/live/node.yourdomain.com/privkey.pem
  upload_limit: 100
system:
  data: /home/daemon-files
  sftp:
    bind_port: 2022
remote: https://panel.yourdomain.com
```

---

## Important Settings

| Setting | Description |
|---------|-------------|
| `api.port` | Wings API port (default: `8080`) |
| `api.ssl.enabled` | Must be `true` in production |
| `api.upload_limit` | Max file upload size in MB |
| `system.data` | Where server files are stored |
| `system.sftp.bind_port` | SFTP access port (default: `2022`) |
| `remote` | Your panel URL |
| `token` | The daemon authentication token |

---

## Regenerating the Token

If you suspect your token is compromised:

1. **Panel Admin → Nodes → your node → Reset Daemon Token**
2. On the Wings server, re-run:

```bash title="bash"
wings configure --panel-url https://panel.yourdomain.com --token NEW_TOKEN
systemctl restart wings
```

---

## Allowed Mount Paths

To allow Docker containers to mount host directories, add them to `config.yml`:

```yaml
allowed_mounts:
  - /mnt/storage
```

---

## Applying Config Changes

After editing `config.yml`:

```bash title="bash"
systemctl restart wings
journalctl -u wings -f  # watch for errors
```
